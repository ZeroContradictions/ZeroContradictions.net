1. Introduction

I always knew that I wanted to be a programmer / software engineer ever since I was 14 years old. When I heard my first explanation of what a computer scientist is and what a computer scientist does, I immediately thought that that would be the perfect job for me when I became an adult. But unfortunately, I encountered numerous obstacles before I finally got to where I wanted to be.

While I’m glad that I finally got where I wanted to be and had all the right environmental conditions by the time I was about 23 years old,

2. Early High School

  • I attended my ninth grade year at a Technical Charter High School.
  • I first started doing HTML and learning really basic programming concepts with Alice in ninth grade when I was 14 years old, but that was hardly close to “real” programming at all.
  • In March 2014, my parents bought me a book called “Learn C++ in 24 Hours”, and it is frankly a horrible book to learn how to program from compared to all the other books out there. I suppose that it was better than nothing, but there are several problems with the book:
    • Many of the things that it says that are great about C++ are very vague and could be applied to other programming languages as well.
    • It didn’t teach how to access library functions, which is the bulk of the actual language. No introductory teaching of a programming language can be considered complete if it doesn’t even teach how to access library functions.
    • It is misguiding to have people believe that you can learn an entire programming language in just a single day, but as I was 14 years old at the time, I didn’t know any better.
    • i
  • I wrote my first programs on an iPad. And I didn’t even have a physical keyboard to type on since my parents couldn’t afford to buy me a real computer since we lived paycheck to paycheck.
    • Since the iPad didn’t have any software that could run a C++ compiler on it, I used an online C++ compiler as a work-around since that could run in the iPad’s Safari web browser on the iPad. That was the only way that I could think of to be able to compile code on a device that was never made for programming to begin with. Sometimes, the work that I typed into the online compiler didn’t get saved, but sometimes it did if the code was saved into as web browser cookies. I only typed really short programs due to the limitations I had.
  • When my grandmother gave my parents the money to buy me a computer, I didn’t even get the computer that I wanted because I was given a 15“ mid-2012 MacBook Pro, when I wanted a computer that ran Linux instead.
    • To be fair, it had great specs for being a first computer, but it still wasn’t what I wanted.
  • I switched my keyboard layout from Qwerty to Dvorak, then to Colemak when I was 16 years old, so I learned how to type on a keyboard three times over.
  • When I first heard of Emacs at 15 years old, I was told that it had a really really high learning curve, but based on the online testimonies of the people that I heard from the people who use it, I knew that that was exactly what I wanted to use as my text editor, but I was mislead into believing that I couldn’t use Emacs on the keyboard that I had due to a phenomenon known as the Emacs Pinky. And while this was true to a great extent, there were some things that could’ve been done to make my life easier.
